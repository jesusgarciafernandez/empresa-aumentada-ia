# Skill: Generación de Checklists

**Objetivo:** Crear listas de tareas operativas para los gestores, ordenadas y completas.

## Cuándo se activa
- Se abre un nuevo caso o incidencia.
- Se va a gestionar una renovación o recurrencia.
- Se da de alta un cliente nuevo (onboarding).
- Hay un cambio sustancial en un servicio o contrato.

## Entrada
- Tipo de trámite (Caso/Incidencia, Renovación, Alta…).
- Producto o servicio asociado.

## Salida
1. Datos necesarios a recopilar.
2. Documentos a solicitar.
3. Acciones internas a realizar (ej. "Registrar en el CRM").
4. Comunicaciones a realizar (cliente / proveedor).
5. Validaciones a comprobar.
6. Riesgos operativos a vigilar.

## Límites
No ejecuta las tareas, solo genera la guía paso a paso. Usa `plantillas/checklist_proceso.md` como base.

## Ejemplo
*Usuario:* "Checklist para gestionar una devolución."
*Skill:* Lista estructurada desde la recepción de la solicitud hasta el cierre y registro.
